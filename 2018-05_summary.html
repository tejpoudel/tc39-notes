Summary for the 64th meeting of Ecma TC39

Ecma Technical Committee 39 held three day meeting at Bloomberg in NYC on May 22nd - May 24th, 2018. Agenda can be found here (note: committee has adopted new agenda formart based on the stage of proposals)

New Proposals

TC39 discussed 10 new proposals. 7 entered staging process to discuss towards specification, and 3 remains at Stage 0 for further investigation before consensus.

Advanced to Stage 1

  • Sequence properties in Unicode property escapes
  • Module Keys
  • Class Static Block
  • Class Access Expressions
  • Pattern Matching
  • RegExp Match array offsets

Advanced to Stage 2

  • Well-formed JSON.stringify

Needs more discussion (remains Stage 0)

  • Tagged Collection Literals
  • Symbol.thenable
  • "Blöcks" syntax

Existing Proposals

Advanced to Stage 2

  • Object.fromEntries
  • Set Methods
  • Realms
  • Function.prototype.toString() censorship. Please also refer to continued discussion
  • Top Level Await

Advanced to Stage 3

  • Symbol.prototype.description
  • Static class features

Advanced to Stage 4 (will be added to ES2019)

  • Optional catch binding
  • ECMAScript as a superset of JSON

Demoted to Stage 2

A conflict with other proposal was found, champion group and committee has decided to move following proposal from Stage 3 to Sage 2. Please see notes for more information.

  • Numeric separators.

Needs more discussion/information

These proposals were discussed but did not advance to next stage at the meeting. Please refer to "Conclusion / Resolution" section on the notes to see the outcome of discussion.

  • Binary AST
  • Function.prototype.toString revision
  • String.prototype.codePoints
  • Class fields
